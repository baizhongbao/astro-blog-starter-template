## 项目简介

分层记忆 + 增量更新 + KV Cache 优化的写作续写工具。

## 特点

- **分层记忆**：概要（云端总结）+ 近期文本（本地缓存），避免每次喂全文
- **增量更新**：每新增 1000 字自动触发概要更新
- **双模型协作**：云端模型总结概要，本地模型生成续写
- **风格一致**：云端模型分析并记录写作风格，续写保持一致
- **KV Cache 优化**：后台服务模式，复用缓存，秒级响应

## 文件结构

```
write_continue/
├── 续写助手.bat      # 续写交互入口
├── 启动服务.bat      # 启动后台服务（KV Cache 模式）
```

## 快速开始

### 1. 配置 API Key

复制 `.env.example` 为 `.env`，填入你的 API Key。

### 2. 启动后台服务（推荐）

双击 `启动服务.bat`，等待服务启动完成（约 5-10 秒），显存占用约 3-4GB。

### 3. 开始续写

双击 `续写助手.bat` → 选择文章 → 生成续写

### 4. 完成后释放显存

双击 `停止服务.bat`

## 使用模式

### 模式一：后台服务模式（推荐）

适合连续续写，KV Cache 复用，秒级响应。

- 续写秒响应
- KV Cache 自动复用
- 服务运行时持续占用 3-4GB 显存

### 模式二：独立进程模式（兼容回退）

适合偶尔续写，显存自动释放。当后台服务未运行时自动回退。

- 无需管理后台服务
- 显存用完即释放
- 每次启动有 5-10 秒延迟

## 技术细节

### 概要结构（云端生成）

**小说**：
```
## 人物
- 张三：主角，性格xxx

## 设定
- 世界观：xxx

## 当前剧情
- 已完成：xxx

## 伏笔
- 未解决：xxx

## 风格
细腻/简洁/幽默/抒情
```

### 双模型协作

| 任务 | 模型 | 原因 |
|------|------|------|
| 概要总结 | 云端（MiMo Pro/DeepSeek） | 需要强理解能力 |
| 续写生成 | 本地（Qwen3-4B） | 省钱，速度快 |

### KV Cache 优化

**原理**：
- 无缓存：每次生成都要重新处理上下文
- 有缓存：已处理的内容缓存起来，只处理新增部分

**显存影响**：
- 模型权重：~2.5GB（固定）
- KV Cache：每 1000 字约 0.5GB

### 自动模式切换

程序会自动检测后台服务状态：
1. 如果服务运行 → 使用 HTTP API（KV Cache 模式）
2. 如果服务未运行 → 回退到独立进程模式

## 注意事项

1. 确保已安装 Python 3.10+ 和 requests 库
2. 本地模型需要 CUDA 支持
3. 云端模型需要 API 密钥
4. 首次续写会自动生成概要
5. API Key 请配置在 `.env` 文件中，不要提交到 Git
